Сервис по проверке доступности одного приложения с другим.

Имеющиеся проекты:
 * KeyChecker.Domain - библиотека основных понятий сервиса
 * KeyChecker.Application - основная библиотека выполняющая логику проверки ключа доступа
 * KeyChecker.Infrastructure.Stubs - реализации-заглушки инфраструктурных репозиториев
 * KeyChecker.Infrastructure.TestImplementation - реализации инфраструктурных репозиториев в памяти приложения с заполнением нескольких тестовых значений
 * KeyChecker.Xunit.Tests - юнит-тесты методов приложения
 * KeyChecker.ConsoleTestApp - использование сервиса валидации в консоли. Так же дёргает апи через апи клиент
 * KeyChecker.Api - апи, которое используя KeyChecker.Application, осуществляет проверку валидации ключа приложения
 * KeyChecker.Api.Client - позволяет быстро осуществлять запросы к KeyChecker.Api 
 * KeyChecker.Api.Models - библиотека моделей запросов к KeyChecker.Api

Основные методы сервиса
 * ValidateKeyAsync - проверяет правильность ключа доступа между приложением и целевым приложением. Ключ должен быть включён.
 * GetAllKnownApplicationsAsync - получает список всех известных сервису приложений
 * GetPermittedApplicationsAsync - для указанного целевого приложения получает список приложений имеющих к нему доступ

Некоторые понятия:
 * Приложение (запрашивающее приложение) - приложение пытающееся получить доступ к другому
 * Целевое приложение - приложение к которому пытаются получить доступ и чей ресурс использовать
 * Идентификация приложений - осуществляется по коду приложения(строка) или идентификатору(guid)
 * Ключ аутентификации - значение (строка) предоставляющее доступ между приложением и целевым приложением. Между одними и теми же приложениями и целевыми приложениями может быть только один ключ. Ключ может быть включен и выключен, только включенный ключ позволяет получить доступ к целевому приложению.

Инфраструктура:
 * Репозиторий приложений - осуществляет методы получения данных о приложениях
 * Репозиторий ключей - осуществляет методы получения данных о ключах
